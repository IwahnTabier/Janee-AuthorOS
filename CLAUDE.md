# CLAUDE.md

## Project Overview

This repository contains the Author Operating System (Author OS) for Janee Butterfield.

Janee Butterfield is a horror and thriller author.

Published works include:

* Caught in Cryptic
* Falling Cryptic
* Nighty Night Dear

The purpose of this repository is to help build Janee's author platform, increase industry visibility, grow readership, and ultimately support securing a traditional publishing deal.

---

## Primary Business Objectives

Priority order:

1. Land a literary agent.
2. Land a traditional publishing deal.
3. Build a recognizable author brand.
4. Grow an email subscriber base.
5. Increase book sales and reviews.
6. Create a sustainable path toward becoming a full-time author.

All technical decisions should support one or more of these objectives.

## Technical Architecture

Preferred stack:

* Python
* SQLite
* PostgreSQL (future)
* Docker
* Git

Favor simplicity over complexity.

Avoid introducing unnecessary frameworks.

Avoid microservices unless clearly justified.

---

## Security Requirements

Assume repository owner is an experienced cybersecurity professional.

Requirements:

* Least privilege.
* Human approval before destructive actions.
* No automatic credential storage.
* No automatic posting to social media.
* No automatic email sending without explicit approval.
* No browser automation without approval.
* No exposure of secrets in source control.

Never commit API keys.

Never create public services unless explicitly requested.

Prefer local execution whenever possible.

---

## Development Standards

* Create documentation for all major features.
* Keep code maintainable.
* Use clear naming conventions.
* Create unit tests when practical.
* Favor readability over cleverness.

Before major changes:

* Explain proposed approach.
* Identify risks.
* Identify dependencies.

---

## Reporting Standards

Reports should prioritize actionable opportunities.

Rank opportunities based on:

* Audience relevance
* Genre fit
* Accessibility
* Potential reach
* Publishing value

Output should be concise and decision-oriented.
